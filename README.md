# รายงานสรุปภาพรวมและสถาปัตยกรรมของโครงการ Tatuga School (Tatuga School Project Summary & Architecture Report)

## 1. บทนำและภาพรวมของระบบ (Introduction & Project Overview)

Tatuga School เป็นแพลตฟอร์มและเว็บแอปพลิเคชันสำหรับการบริหารจัดการการเรียนรู้ (Learning Management System - LMS) ที่ถูกพัฒนาขึ้นมาเพื่อแก้ไขปัญหาและตอบสนองความต้องการของการจัดการศึกษาในยุคดิจิทัลอย่างครบวงจร แพลตฟอร์มนี้ถูกออกแบบมาโดยคำนึงถึงความสะดวกสบายของทั้งผู้สอน (ครู) และผู้เรียน (นักเรียน) เป็นหลัก โดยมีเป้าหมายในการลดภาระงานเอกสารของครู และเพิ่มการมีส่วนร่วมของนักเรียนในชั้นเรียน

หนึ่งในจุดเด่นที่สำคัญที่สุดของระบบนี้คือ **การจัดการเรียนการสอนแบบสมัยใหม่ที่ตัดปัญหาความยุ่งยากทางเทคนิค** ตัวอย่างเช่น ระบบอนุญาตให้นักเรียนสามารถเข้าสู่ระบบและส่งงานได้โดยตรงผ่านรหัสประจำตัว หรือระบบบัญชีภายในของโรงเรียน โดยที่นักเรียนไม่จำเป็นต้องมีที่อยู่อีเมล (Email) ซึ่งตอบโจทย์สำหรับนักเรียนในระดับชั้นประถมศึกษา หรือนักเรียนที่ยังไม่มีความเชี่ยวชาญในการจัดการบัญชีอีเมลส่วนตัว

นอกจากนี้ Tatuga School ยังได้ก้าวไปอีกขั้นด้วยการผสานเทคโนโลยีปัญญาประดิษฐ์ (Artificial Intelligence - AI) เข้ามาเป็นแกนหลักในหลายส่วนของระบบ เพื่อยกระดับประสบการณ์การเรียนรู้และการสอนให้มีประสิทธิภาพสูงสุด ไม่ว่าจะเป็นการใช้ AI ในการวิเคราะห์ข้อมูลนักเรียนเพื่อแนะนำเส้นทางอาชีพในอนาคต (Career Suggestion) การจัดทำคลังสื่อการสอนที่รองรับการค้นหาเชิงความหมาย (Semantic Search) ด้วย AI ตลอดจนการทำงานร่วมกับแพลตฟอร์มโซเชียลยอดนิยมอย่าง Line เพื่อสร้างระบบแจ้งเตือนและแชทบอทถาม-ตอบที่เปรียบเสมือนผู้ช่วยครูประจำวิชา

ฟีเจอร์เด่นที่ครอบคลุมการทำงานของระบบ ได้แก่:

- **ระบบมอบหมายและจัดการส่งงาน (Assignment Management):** รองรับทั้งการส่งงานแบบข้อความและไฟล์ข้อมูล และยังมีฟีเจอร์ Video Quiz ที่สามารถบังคับไม่ให้นักเรียนกดข้าม (Prevent Fast-forward) และแทรกคำถามตามเวลา (Timestamp) ของวิดีโอได้
- **ระบบวิเคราะห์และแนะนำอาชีพด้วย AI (AI-Driven Career Suggestion):** ระบบจะทำการรวบรวมทักษะ (Skills) ที่นักเรียนได้รับจากการทำชิ้นงานต่างๆ มาวิเคราะห์ทางสถิติเปรียบเทียบกับค่าเฉลี่ยของกลุ่มประชากรนักเรียนทั้งหมด แล้วจับคู่เข้ากับฐานข้อมูลอาชีพโดยใช้เทคโนโลยี Vector Search
- **คลังสื่อการสอนและชุมชนแห่งการแบ่งปัน (Teaching Material Hub):** พื้นที่สำหรับครูในการแบ่งปันสื่อการสอน โดยระบบสามารถดึงข้อมูล Meta Tags จากลิงก์ภายนอกได้อัตโนมัติ สร้างรูปภาพ Thumbnail จากไฟล์ PDF และรองรับการค้นหาด้วยปัญญาประดิษฐ์
- **ระบบจัดการการเข้าเรียน (Attendance Tracking System):** รองรับการเช็คชื่อแบบปกติ และการเช็คชื่อผ่านระบบสแกน (Scan) ซึ่งสามารถกำหนดเวลาหมดอายุ และการอนุญาตให้สแกนซ้ำได้
- **การจัดการทำงานร่วมกัน (Team & Kanban Board):** มีระบบการจัดการบอร์ดงานแบบ Kanban (Board, Column, Task) ภายในระบบ เพื่อให้บุคลากรหรือนักเรียนทำงานร่วมกันเป็นทีมได้อย่างมีประสิทธิภาพ
- **การแจ้งเตือนและการตอบคำถามอัจฉริยะ (Line Bot & Smart Notifications):** ระบบจะส่งการแจ้งเตือนแบบเรียลไทม์ไปยัง Line Group เมื่อมีการส่งงาน พร้อมด้วย AI ที่สามารถทำหน้าที่เป็นผู้ช่วยในการตอบคำถามทางวิชาการที่เกี่ยวข้องกับเนื้อหาในรายวิชา

## 2. โครงสร้างและการทำงานของส่วนหน้าจอผู้ใช้งาน (Client-Side Architecture & Workflow)

สถาปัตยกรรมฝั่งไคลเอนต์ (Frontend Architecture) ของโครงการ Tatuga School ถูกออกแบบมาในรูปแบบแยกส่วนประกอบ (Decoupled/Micro-frontend Concept) โดยแบ่งการทำงานออกเป็น 3 แอปพลิเคชันหลักอย่างชัดเจน เพื่อแยกระดับสิทธิ์การเข้าถึง (Authorization) และมอบประสบการณ์ผู้ใช้ (User Experience - UX) ที่ดีที่สุดสำหรับกลุ่มเป้าหมายแต่ละกลุ่ม:

### 2.1 ระบบหน้าหลัก (Homepage - `client-homepage-tatugaschool`)

แอปพลิเคชันส่วนนี้ทำหน้าที่เป็นหน้าด่านแรก (Landing Page) สำหรับการแนะนำแพลตฟอร์ม การทำการตลาด และดึงดูดผู้ใช้งานใหม่ให้เข้ามาสมัครใช้บริการ

- **เทคโนโลยีหลัก:** พัฒนาด้วย **Next.js 14** ซึ่งเน้นการทำ Static Site Generation (SSG) และ Server-Side Rendering (SSR) เพื่อประสิทธิภาพในการโหลดหน้าเว็บที่รวดเร็ว และผลลัพธ์ที่ดีในด้าน SEO (Search Engine Optimization)
- **ระบบจัดการเนื้อหา (CMS):** มีการนำ **Sanity CMS** เข้ามาบูรณาการ (`next-sanity`, `@portabletext/react`) ทำให้ทีมงานการตลาดหรือแอดมินสามารถอัปเดตบทความ ข่าวสาร หรือเนื้อหาบนหน้าเว็บได้ทันทีโดยไม่ต้องแก้ไขโค้ด

### 2.2 ระบบสำหรับครูและผู้ดูแลโรงเรียน (Main Application - `client-main-tatugaschool`)

เป็นระบบที่มีความซับซ้อนและขนาดใหญ่ที่สุดในโครงการ ทำหน้าที่เป็นแผงควบคุม (Dashboard) สำหรับผู้บริหารโรงเรียนและบุคลากรครู

- **เทคโนโลยีหลัก:** พัฒนาด้วย **Next.js 16** ซึ่งเป็นเวอร์ชันใหม่ล่าสุดที่ให้ประสิทธิภาพการเรนเดอร์ระดับสูง
- **ระบบการจัดการ State และ API:** มีการใช้ **React Query (TanStack Query v5)** อย่างเต็มรูปแบบร่วมกับ Axios เพื่อการจัดการ Caching ข้อมูลจากเซิร์ฟเวอร์ การจัดการสถานะของ UI ระหว่างการโหลด และการทำ Optimistic Updates
- **เครื่องมือ UI ระดับสูง:**
  - **Drag and Drop:** มีการใช้ `@dnd-kit` เพื่อสร้างระบบกระดาน Kanban Board สำหรับการจัดการงานต่างๆ แบบลากวาง
  - **Rich Text Editor:** ใช้ `tinymce` เพื่อให้ครูสามารถเขียนรายละเอียดรายวิชาหรือชิ้นงานได้อย่างอิสระเหมือนใช้งานโปรแกรม Word
  - **Data Visualization:** มีการใช้ `chart.js` ผ่าน `react-chartjs-2` เพื่อแสดงผลสถิติการเรียนของนักเรียน สถิติการส่งงาน ในรูปแบบกราฟที่เข้าใจง่าย
  - **Export to PDF/Image:** รองรับการแปลงผลลัพธ์หน้าจอเป็นรูปภาพหรือเอกสาร PDF ผ่าน `html2canvas` และ `jspdf`
  - **Payment & Subscriptions:** ผสานการทำงานร่วมกับ `@stripe/react-stripe-js` เพื่อการชำระเงินค่าแพ็กเกจ (Subscription Plan) ของโรงเรียน ภายในระบบโดยตรง

### 2.3 ระบบสำหรับนักเรียน (Student Application - `client-student-tatugaschool`)

แอปพลิเคชันที่ถูกปรับแต่งมาเพื่อให้มี User Interface ที่เป็นมิตรและเข้าใจง่ายสำหรับนักเรียน

- **เทคโนโลยีหลัก:** พัฒนาด้วย **Next.js 15.1**
- **การทำงานหลัก:** นักเรียนสามารถเข้าสู่ระบบเพื่อดูรายวิชาของตนเอง ตรวจสอบงานที่ได้รับมอบหมาย ทำแบบทดสอบ (Video Quiz) และส่งงาน ไม่ว่าจะเป็นในรูปแบบของการพิมพ์ข้อความ (ผ่าน `tinymce`) หรือการอัปโหลดไฟล์ นอกจากนี้ยังมีระบบ Countdown Timer (`react-countdown`) สำหรับการสอบหรืองานที่มีการจำกัดเวลา

**สรุป Tech Stack และการออกแบบ UI ของทั้งฝั่ง Client:**
ส่วนของการออกแบบ UI ของทุกแอปพลิเคชันอยู่บนมาตรฐานเดียวกัน โดยใช้ **TailwindCSS** คู่กับ **Styled Components** สำหรับการจัดการ CSS องค์ประกอบของ UI (Component Library) นำมาจาก **PrimeReact** เพื่อความรวดเร็วและเป็นมาตรฐาน และมีการใช้ **Framer Motion** และ **Canvas-Confetti** เพื่อเพิ่ม Animation และ Interactive feedback ช่วยสร้างประสบการณ์ใช้งานที่น่าตื่นเต้น

## 3. โครงสร้างและการทำงานของส่วนเซิร์ฟเวอร์ (Server-Side Architecture & Workflow)

ระบบหลังบ้าน (Backend Server) คือหัวใจหลักที่คอยควบคุมและจัดการข้อมูลลอจิก (Business Logic) ทั้งหมดของแพลตฟอร์ม ระบบนี้อยู่ในไดเรกทอรี `server-main-tatugaschool`

- **Framework ประสิทธิภาพสูง:** พัฒนาด้วยสถาปัตยกรรมโมดูล (Modular Architecture) ของ **NestJS 11** ทำให้โค้ดถูกแบ่งออกเป็นสัดส่วน (Controller, Service, Repository, Module) ง่ายต่อการขยายตัว (Scalability) และการบำรุงรักษา
- **JavaScript Runtime แห่งอนาคต:** ระบบเซิร์ฟเวอร์ไม่ได้รันบน Node.js ทั่วไป แต่ถูกตั้งค่าให้รันบน **Bun** (เห็นได้จากคำสั่ง `bun dist/main.js` ใน package.json) ซึ่งให้ประสิทธิภาพการทำงานที่สูงกว่ามาก ทั้งในด้านระยะเวลาการเปิดตัว (Start-up time) และการจัดการหน่วยความจำ
- **Security & Authorization:**
  - การจัดการสิทธิผู้ใช้งานมีความซับซ้อน โดยแบ่งตั้งแต่ Admin, School Billing Manager, Teacher, ไปจนถึง Student
  - ระบบตรวจสอบตัวตน (Authentication) ใช้ **Passport.js** รองรับการ Login ผ่าน Local Strategy (Username/Password), JWT Token (แยกคีย์ระหว่างบุคคลทั่วไปและนักเรียน), และ Google OAuth2
- **Background Processes & Caching:**
  - นำ **Redis (ioredis)** เข้ามาช่วยในเรื่องของ Caching ข้อมูลเพื่อลดภาระของฐานข้อมูล
  - ใช้ `@nestjs/schedule` สำหรับสร้าง Cron Jobs เพื่อประมวลผลงานเบื้องหลัง เช่น การล้างข้อมูล การคำนวณสถิติรายวัน หรือการแจ้งเตือนล่วงหน้า
- **File Management & Document Parsing:**
  - รองรับการประมวลผลไฟล์เอกสาร PDF (`pdf-parse`) และ Excel (`exceljs`) บนเซิร์ฟเวอร์โดยตรง
  - มีระบบการบีบอัดไฟล์ (Archiver) ก่อนส่งให้ผู้ใช้ดาวน์โหลด
  - ใช้งานแพ็กเกจ Image Processing (`canvas`, `blurhash`) ในการแปลงรูปภาพ หรือสกัดสีและโครงสร้างภาพมาทำ Blurhash string ช่วยให้เวลาที่แอปพลิเคชันโหลดภาพ จะแสดงสีสันเป็นโครงร่างเบลอๆ ขึ้นมาก่อน ลดความหงุดหงิดจากการรอคอยของผู้ใช้

## 4. การจัดการฐานข้อมูลและสถาปัตยกรรมข้อมูล (Database Architecture & Data Modeling)

Tatuga School เลือกใช้ **MongoDB** เป็นฐานข้อมูลหลัก (Database Provider) โดยเชื่อมต่อผ่าน **Prisma ORM** ซึ่งถือเป็นการผสมผสานที่ทรงพลังมาก เนื่องจาก MongoDB ให้ความยืดหยุ่นสูงในฐานะ NoSQL แต่การใช้ Prisma ช่วยบังคับโครงสร้างและความสัมพันธ์ (Relational Data Modeling) ของข้อมูลให้มีความรัดกุม

โครงสร้างตาราง (Models) ที่สำคัญภายใน `schema.prisma` แสดงให้เห็นถึงขนาดที่ใหญ่ของระบบ:

- **Core Entities:** `User`, `School`, `Student`, `Class`, `Team`
- **Academic Setup:** `Subject`, `TeacherOnSubject`, `StudentOnSubject`, `GroupOnSubject`
- **Assignment System:** ระบบงานไม่ได้มีแค่งานแบบปกติ แต่มีสถานะ (`Draft`, `Published`), ชนิด (`Assignment`, `VideoQuiz`, `Material`), การบันทึก Vector ของงาน (`vector`) และโครงสร้างคำถามพร้อม Timestamp สำหรับ `QuestionOnVideo`
- **Grading & Scoring System:** ตาราง `ScoreOnSubject`, `ScoreOnStudent`, และ `GradeRange` (ซึ่งเก็บข้อมูลการตัดเกรดเป็น JSON Format)
- **Attendance Management:** ประกอบด้วยตารางระดับโครงสร้าง เช่น `AttendanceTable`, ระดับแถว `AttendanceRow` (สามารถตั้งค่ากำหนดเวลาหมดอายุและการสแกนได้) และข้อมูลการเข้าเรียนรายคน `Attendance`
- **Skills & Career Mapping:**
  - มีโมเดลระดับทฤษฎี (Theory Model) เช่น `Skill`, `Career`, `SkillOnCareer` (ความสัมพันธ์แบบ Many-to-Many พร้อมตัวคูณน้ำหนักหรือ weight)
  - และระดับการนำไปใช้จริงคือ `SkillOnAssignment` (การผูกทักษะเข้ากับชิ้นงาน) และ `SkillOnStudentAssignment` (ทักษะตกผลึกไปยังนักเรียนตามน้ำหนักคะแนนชิ้นงานที่นักเรียนได้)

## 5. ระบบปัญญาประดิษฐ์และฟีเจอร์ขั้นสูง (Artificial Intelligence & Advanced Features)

ความแตกต่างของ Tatuga School กับระบบ LMS ทั่วไป คือการนำ AI อย่าง **Google GenAI (Gemini)** มาเป็นฟันเฟืองขับเคลื่อน (Engine) หลักในหลายฟีเจอร์ ภายในไฟล์ `ai.service.ts` และเซอร์วิสอื่นๆ:

### 5.1 ระบบวิเคราะห์และแนะนำอาชีพอัจฉริยะ (AI-Driven Career Suggestion)

ระบบทำงานในคลาส `CareerService` อย่างล้ำลึก:

- **Data Aggregation:** ระบบจะไปดึงคะแนนของ `SkillOnStudentAssignment` ทั้งหมดของนักเรียน มาหาค่าเฉลี่ยทักษะรายบุคคล
- **Population Comparison:** จากนั้นจะนำคะแนนดังกล่าวไปเปรียบเทียบกับกลุ่มประชากร (นักเรียนทั้งหมด) เพื่อหาเปอร์เซ็นไทล์ว่านักเรียนคนนี้มีคะแนน "สูงกว่า (above)" หรือ "ต่ำกว่า (below)" คนอื่นกี่คน
- **Match Point Calculation:** นำค่าความชำนาญของนักเรียน มาคำนวณความสอดคล้องกับแต่ละอาชีพ (Career) ออกมาเป็นค่า `careerMatchPoint` เรียงลำดับอาชีพที่เหมาะสมจากมากไปน้อย ทำให้นักเรียนค้นพบศักยภาพที่แท้จริงของตนเองได้อย่างแม่นยำ
- **AI Embedding:** เมื่อผู้สอนสร้างหรือแก้ไขข้อมูลอาชีพ AI จะนำ ชื่อ (Title), คำอธิบาย (Description), และคำค้นหา (Keywords) มาประมวลผลเป็นคณิตศาสตร์เวกเตอร์ (Vector Float Array) เพื่อการค้นหาเชิงความหมาย

### 5.2 ระบบจัดการและค้นหาสื่อการสอน (Teaching Material System)

ระบบในคลาส `TeachingMaterialService` ถูกออกแบบมาเพื่ออำนวยความสะดวกในการจัดเก็บสื่อการสอน:

- **Semantic Search:** เมื่อครูค้นหาสื่อการสอน ระบบจะนำประโยคค้นหานั้นไปแปลภาษา (Translation) ก่อน จากนั้นใช้ AI คำนวณเป็น Vector และทำการเปรียบเทียบเวกเตอร์ด้วยคำสั่งค้นหาเวกเตอร์ใน MongoDB
- **Metadata Auto-Extraction:** เมื่อครูแปะลิงก์ของสื่อจากภายนอก เซิร์ฟเวอร์จะใช้ `cheerio` ไปอ่านข้อมูลหน้าเว็บ (Web Scraping) เพื่อดึง Meta Tags `og:title`, `og:description`, `og:image` ออกมาแสดงเป็นภาพพรีวิวอัตโนมัติ
- **PDF Thumbnail Generator:** เซิร์ฟเวอร์สามารถรันการทำงานแปลงหน้าแรกของไฟล์เอกสาร PDF ให้ออกมาเป็นรูปภาพ (Thumbnail) โดยไม่ต้องพึ่งพาแอปพลิเคชันอื่น

### 5.3 ระบบผู้ช่วยและการแจ้งเตือนผ่าน Line (Line Bot & AI Assistant)

ในคลาส `LineBotService` ผนวกกับการใช้งานแพ็กเกจ `@line/bot-sdk`:

- เมื่อนักเรียนอัปโหลดไฟล์ส่งงาน หรือมีการเปลี่ยนแปลงสถานะในชั้นเรียน ระบบสามารถ Push Message แจ้งเตือนไปยังกลุ่มไลน์ (Line Group) ของผู้ปกครองหรือนักเรียนได้ทันที
- และยังสามารถทำงานร่วมกับ AI เพื่อวิเคราะห์ข้อความที่ถามเข้ามาใน Line (Reply Message) โดยอ้างอิงจากบริบทข้อมูล (Context) ของรายวิชานั้นๆ ช่วยแบ่งเบาภาระการตอบคำถามซ้ำๆ ของผู้สอนได้เป็นอย่างดี

## 6. การบูรณาการกับระบบและบริการภายนอก (Third-Party Integrations)

เพื่อให้ระบบมีความปลอดภัย มีเสถียรภาพ และทำงานได้สมบูรณ์แบบ Tatuga School มีการเชื่อมต่อกับระบบคลาวด์และ API ภายนอกมากมาย:

1. **Amazon Web Services (AWS S3):** ควบคุมและดูแลพื้นที่เก็บข้อมูล (Storage) สำหรับไฟล์รูปโปรไฟล์ ไฟล์งานที่นักเรียนส่ง และสื่อการสอนที่มีขนาดใหญ่ โดยสร้าง Presigned-URL ในการเข้าถึงไฟล์อย่างปลอดภัย
2. **Stripe Payment Gateway:** โครงสร้างข้อมูล `School` มีการผูกกับ `stripe_customer_id`, `stripe_price_id` และกำหนด Limit ต่างๆ เช่น จำกัดจำนวนชั้นเรียน จำกัดความจุไฟล์ ระบบจึงสามารถตัดบัตรเครดิตและอัปเกรดสถานะโรงเรียนได้อย่างอัตโนมัติผ่านระบบ Webhook ของ Stripe
3. **Communication Services:**
   - **Nodemailer & Mailersend:** สำหรับการส่งอีเมลเพื่อยินยันตัวตน (Verify Email) หรือแจ้งรีเซ็ตรหัสผ่าน (Reset Password)
   - **Web Push Notifications:** รองรับการแจ้งเตือนตรงสู่เบราว์เซอร์ของผู้ใช้ ผ่านแพ็กเกจ `web-push` และ VAPID Keys

## 7. กระบวนการพัฒนา และ DevOps (Development Workflow & CI/CD)

นอกจากตัวแอปพลิเคชัน โครงการนี้ยังมีมาตรฐานทางวิศวกรรมซอฟต์แวร์ (Software Engineering) ที่ยอดเยี่ยม:

- **Containerization (Docker):** ระบบมี `Dockerfile` สำหรับการตั้งค่า Environment ทั้งในโหมดพัฒนา (Development), โหมดทดสอบ (Test), และโหมด Production การใช้งาน `docker-compose` ช่วยให้นักพัฒนาสามารถจำลองฐานข้อมูล MongoDB ของตัวเองขึ้นมาได้อย่างรวดเร็ว (`db:dev:up`)
- **Automated Testing:** มีการวางโครงสร้างระบบทดสอบแบบครบวงจร (เห็นได้จากไฟล์ `.spec.ts` ตามจุดต่างๆ) โดยใช้ **Jest** ในการทำ Unit Test และใช้ **Supertest** ในการทำ End-to-End Test
- **Continuous Integration/Continuous Deployment (CI/CD):** ภายใน `package.json` มีสคริปต์ที่จัดการความยุ่งยากเหล่านี้แบบอัตโนมัติ เช่น `docker:build:run:test` ซึ่งเป็นการสั่งให้สร้าง Image จากคอนเทนเนอร์ นำไปเปิดเซิร์ฟเวอร์จำลอง และรัน Test Script เข้าไปทดสอบเพื่อหาข้อบกพร่อง ก่อนที่จะอนุญาตให้ปล่อยระบบเวอร์ชันใหม่สู่ผู้ใช้งานจริง

โปรเจกต์ Tatuga School ไม่ได้เป็นเพียงเว็บไซต์จัดเก็บเอกสารประกอบการเรียน แต่เป็น **EdTech Platform (แพลตฟอร์มการศึกษาทางเทคโนโลยี)** ที่มีความสลับซับซ้อนและเปี่ยมไปด้วยนวัตกรรม ทั้งสถาปัตยกรรม Micro-frontend ด้วย Next.js ตัวหลังบ้านพลังสูงด้วย NestJS/Bun บนฐานข้อมูล MongoDB และที่โดดเด่นเหนือคู่แข่งในตลาด คือความพยายามอย่างเป็นรูปธรรมในการนำ AI ขั้้นสูงมาประยุกต์ใช้ในการวัดผล ประเมินผู้เรียน และช่วยเหลือบุคลากรทางการศึกษาให้ทำงานได้สะดวกรวดเร็วยิ่งขึ้น
